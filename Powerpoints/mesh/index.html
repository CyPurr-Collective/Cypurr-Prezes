Graph Theory is the branch of mathematics that deals with things like maps, networks, flow, and centrality. Like most of mathematics, it sbtarts with a few basic building blocks that lead to some surprising results.

 

This is a node. It is a basic building block of graph theory. It can be represented as a dot, point, or area on a page. A collection of one or more nodes is called a graph. While it seems abstract, nodes can be used to represent access points on a wireless network, species in an ecosytem, or stops along a supply chain. We will focus on the first such type of node.

 
This is an edge. It connects two nodes. In networks, we call this a link and it allows two nodes to talk.
 

This is a graph. It is a collection of more than one node. In a network, two nodes can be connected as in the figure below. That connection is called a vertex or edge. This is an unconnected graph, which means that some of the nodes cannot talk to each other.

 

Networks

A network is a series of nodes and edges that can be represented by something similar to the above image.

This image in particular a fully-connected network or a complete graph.

Branch - a node that gives and receives services on a network (your ISP)

 

Types of Nodes

Leaf - a node that only receives services on a network (you)

Hub - a node that provides services on a network (a datacenter or internet company)

  
 
This is a path. It connects leaf nodes and branch nodes along a single-path network.
 
Path graphs, while useful for covering large distances cheaply, are not resilient.

A single node removal can change the entire structure of the network.

In some cases, a single point (the yellow node) can act as a local hub. This is called a star graph and can be called as above. These types of networks have the same problems as path graphs.

Your Internet Service Provider (ISP)
Your ISP gets Internet from a small number of datacenters and distributes the resources as they see fit, often incorporating both star and path networks in their designs.
